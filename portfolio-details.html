Requerimientos

Requerimientos Funcioneales

  • Nombre: Registro de usuarios

  • Codigo: Rf 001

  • Prioridad: Alta

Descripcion:

Se debe permitir el registro de cada uno de los usuarios de la aplicación web, permitiéndoles escoger su rol ya sea artista o empresa.


  • Nombre: Consulta de usuarios

  • Codigo: Rf 002

  • Prioridad: Alta

Descripcion :

Se debe permitir la consulta de cada uno de los usuarios de la aplicación web


  • Nombre: Actualización de usuarios

  • Codigo: Rf 003

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir al usuario la actualización de sus datos básicos consignados en la aplicación web


  • Nombre: Eliminación de usuarios

  • Codigo: Rf 004

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir al usuario la eliminación (cambio de estado) únicamente de su usuario


  • Nombre: Cargue de archivos

  • Codigo: Rf 005

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios registrados subir diferentes archivos de tipo imagen, vídeo y audio al aplicativo web.


  • Nombre: Consulta de archivos

  • Codigo: Rf 006

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios registrados visualizar y reproducir todo tipo de imágenes, vídeos y audios cargados por él o por otro usuario.


  • Nombre: Actualización de archivos

  • Codigo: Rf 007

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios registrados actualizar todo tipo de imágenes, vídeos y audios cargados únicamente por él.


  • Nombre: Eliminación de archivos

  • Codigo: Rf 008

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios registrados eliminar todo tipo de imágenes, vídeos y audios cargados únicamente por él.


  • Nombre: Crear proyecto

  • Codigo: Rf 009

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios tipo empresa crear proyectos


  • Nombre: Consultar proyecto

  • Codigo: Rf 0010

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios sin importar su tipo consultar los proyectos creados en la aplicación web


  • Nombre: Actualizar proyecto

  • Codigo: Rf 0011

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios de tipo empresa actualizar los proyectos únicamente creados por él.


  • Nombre: Eliminar proyecto

  • Codigo: Rf 0012

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios de tipo empresa eliminar los proyectos únicamente creados por él.


  • Nombre: Contactar con artista

  • Codigo: Rf 0013

  • Prioridad: Alta

    • Descripcion:

    Se debe permitir a los usuarios de tipo empresa contactar con los usuarios de tipo artista por medio del aplicativo web

    Requerimientos No Funcioneales

    • Nombre: Usabilidad

    • Codigo: Rnf 001

    • Prioridad: Media

    Descripcion:

    La aplicación web debe ser fácil de usar para el usuario.


    • Nombre: Tiempo de respuesta

    • Codigo: Rnf 002

    • Prioridad: Media

    Descripcion:

    La aplicación web debe proporcionar tiempos de respuesta rápidos.


    • Nombre: Accesibilidad

    • Codigo: Rnf 003

    • Prioridad: Media

    Descripcion :

    La aplicación web desarrollada está pensada para que sea accesible utilizando un navegador estándar.


  • Nombre: Control de ataques

  • Codigo: Rnf 004

  • Prioridad: Media

    • Descripcion:

    Si se identifican ataques de seguridad al sistema, el mismo no continuará operando hasta ser desbloqueado por un administrador de seguridad.


  • Nombre: Interfaces gráficas

  • Codigo: Rnf 005

  • Prioridad: Media

    • Descripcion:

    El sistema debe poseer interfaces gráficas bien formadas y claras.


  • Nombre: Complejidad algorítmica

  • Codigo: Rnf 006

  • Prioridad: Media

    • Descripcion:

    Complejidad algorítmica baja para un alto rendimiento.
